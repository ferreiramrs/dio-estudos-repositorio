[PROJETO] Criação de página na web: Conceitos básicos de HTML


No primeiro módulo da formação de desenvolvedor HTML para Web, o professor Diogo Mainardes explicou sobre os tópicos básicos de desenvolvimento HTML.

Muito se questiona se o HTML é uma linguagem de programação ou não. O Hypertext Mark-up Language - ou em português, linguagem de Marcação de Hipertexto - não é uma linguagem de programação,
pois não tem um algoritmo por trás, ou a proposta em si e promover cálculos ou o processamento de dados de um banco ou servidor.

O HTML é uma linguagem de criação e desenvolvimento de páginas web, seja para usar localmente ou hospedar e usar diretamente na web, através de URLs. Trata-se de um script para montar sites.

Introdução: História da Web

Era uma vez um mundo sem internet...
...até que criaram uma rede mágica que conecta tudo, possibilitando que a Barbie fale com as amigas em todos os lugares do mundo!

Primeiros Computadores

No comecinho, os computadores eram grandões como castelos! Não dava pra ver filmes ou jogar
Eles só faziam contas super difíceis, e um dos criadores foi Charles Babbage, que pensou na ideia de uma máquina que calculasse sozinha!

Como surgiu o HTML

Um moço muito esperto, chamado Tim Berners-Lee, criou um jeitinho mágico de mostrar páginas bonitas, tipo um livro encantado da Barbie — isso se chama HTML!

Grandes Nomes

  • Charles Babbage, o “pai do computador”,
  • Ada Lovelace, que foi a primeira programadora da história,
  • Depois vieram o Tim Berners-Lee, que criou a Web,
  • Alan Turing, que ajudou a ensinar os computadores a pensarem!

O que são Clientes

Clientes são como bonecas Barbie que pedem "Ei, quero ver essa página!" — e o computador a entrega rapidinho, da forma esperada por este cliente (ou usuário que pediu para acessar a página).
Por exemplo, quando entramos em um blog e clicamos nos links, que nos redirecionam para uma parte do site.

Cliente no linguajar Web

No vocabulário da web, o "cliente" não é necessariamente aquele que compra algo (como a boneca que vai até a lojinha da Barbie), ele só pede as coisinhas pra ver — como pedir para ver um look novo na loja da Barbie.

Navegadores

Navegadores são como carros conversíveis que a Barbie usa pra passear pela internet e "visitar os lugares", bem conhecidos por sites ou páginas da web.

Aplicações Web

São programas mágicos que funcionam direto no navegador, como o jogo de salão de beleza da Barbie no computador! (Deve ter no Friv Jogos ainda :D)


Voltar

Dispositivos Móveis

Celulares e tablets são como bolsas mágicas: pequenos, mas cheios de poderes pra acessar a internet onde quiser.

O que são servers?

Servers são como o armário da Barbie — eles guardam tudo dos sites e entregam quando alguém pede.

Sobre servidores

Eles trabalham dia e noite pra mandar páginas, vídeos e imagens pros nossos computadores — igual as fadas ajudantes do castelo!

Hardwares e Softwares em servidores

O hardware é o corpo forte do servidor (tudo que se conecta ao computador fisicamente), e o software é o cérebro esperto que sabe o que fazer (sistema operacional, aplicações e softwares).

Voltar

Tipos de servidores

Tem servidor pra guardar arquivos, outro pra enviar e-mails… Cada um tem uma missão, (assim como as Barbies com diferentes profissões!)

Hospedagem de sites

É como alugar uma casinha encantada pro seu site morar e todo mundo poder visitar.

Linguagens de Programação

São como feitiços! A gente escreve códigos e, plim!: o computador entende, processa as informações e faz as coisas acontecerem - assim o cliente consegue visualizar as páginas em sua máquina.

O que é?

Programar é conversar com o computador num idioma mágico pra ele entender e agir — é como dizer “faça um bolo!” e ele faz! (a partir de um código bem estruturado)

Voltar

Server-Side

É quando o trabalho acontece escondido no castelo do servidor, antes de chegar na tela da Barbie.

Client-Side

Aqui, a mágica acontece na frente da gente! A Barbie vê tudo ganhando vida direto no computador.

HTML não é liguagem de programação

O HTML é só o jeitinho de arrumar a casa. Ele não faz mágica, só deixa tudo lindo e organizadinho!

Introdução ao HTML na prática

Vamos brincar de Barbie arquiteta e construir a casinha (neste caso, o site) com tijolinhos chamados tags (atribuitos e etc) de HTML. Cada uma das tags têm abertura (ex: "< head>") e fechamento (""), exceto tags independentes como "< br>" e "< hr>".

Voltar

Ferramentas utilizadas

Para montar sites, usamos caixinhas de ferramentas mágicas, como editores de texto e navegadores.

Inspetor de Elementos

É uma lupa mágica no navegador! A Barbie usa pra ver como as coisas do site estão feitas por trás. Para acessar o Inspetor no navegador, aperta-se a tecla F12 e voilà.

Estrutura Básica do HTML

Todo site tem uma estrutura, como uma casinha de boneca com teto, parede e chão.
No HTML, a estrutura básica segue "< head>""" (o cabeçalho do site, que geralmente armazena metadados, título e etc), este conteúdo normalmente não está visível para o cliente.
No meio temos a parte do corpo do site (representado pelas tags "< body>"""), e é onde vemos o conteúdo da página.

Falando sobre Tags

Tags são palavras mágicas com < e>. A Barbie usa pra dizer “aqui começa um título” ou “aqui vem uma imagem!”.

Voltar

Atributos das Tags

Atributos são brilhos extras das tags: é como dizer que a imagem é grande ou que o botão é rosa.

Textos

No HTML, a Barbie escreve recadinhos, títulos e histórias para aparecerem nos sites!

Listas Ordenadas e Não Ordenadas

A Barbie ama fazer listas! Algumas são numeradas (pra seguir a ordem, tag "< ol> e ") e outras são só pontinhos fofos (ou listas "não-ordenadas", tags "< ul>" e "").

Links

Links são como portais mágicos. A Barbie clica e — puff: vai pra outro lugar da internet!
